# Codex 대화 맥락 인수인계

이 문서는 여러 PC의 Codex가 같은 채팅을 직접 공유하지 못하는 문제를 보완하기 위한 기준입니다.

## 목표

동기화 대상은 실제 프로젝트 파일이 아니라 아래 내용입니다.

```text
1. 마지막 작업 대화 요약
2. 사용자가 내린 결정
3. 보류한 작업
4. 다음 PC에서 이어받아야 할 질문
5. 실제 프로젝트 루트 확인 필요 여부
```

## 중요한 제한

Codex의 실제 채팅 전체를 GitHub에 자동으로 복사하는 기능은 현재 이 저장소만으로는 구현하지 않습니다.

대신 작업 종료 시 Codex가 대화 내용을 요약해서 `docs/HANDOFF_LOG.md`에 남기고,
다음 PC의 Codex가 그 요약을 읽고 이어받습니다.

## 작업 시작 시 Codex가 해야 할 일

1. `START_HERE.md`를 읽습니다.
2. `docs/GLOBAL_RULES.md`를 읽습니다.
3. `docs/HANDOFF_LOG.md`의 마지막 항목을 읽습니다.
4. 필요한 경우 사용자에게 실제 프로젝트 루트 위치를 묻습니다.
5. 프로젝트 파일을 건드리기 전에 대상 폴더를 확인합니다.

## 작업 종료 시 Codex가 남겨야 할 내용

작업 종료 전 아래 내용을 요약해서 저장합니다.

```text
- 어느 PC에서 작업했는지
- 어떤 주제의 대화를 했는지
- 어떤 결정을 했는지
- 어떤 파일이나 DB는 아직 건드리지 않았는지
- 다음 PC에서 무엇을 먼저 확인해야 하는지
- 실제 프로젝트 루트 확인이 필요한지
```

## 프로젝트 루트 규칙

실제 프로젝트 파일 위치는 자동 추정하지 않습니다.

Codex는 작업 시작 시 필요하면 반드시 사용자에게 아래처럼 확인합니다.

```text
이번에 이어서 작업할 실제 프로젝트 폴더가 어디인가요?
예: C:\Users\...\출석프로그램
```

프로젝트별 저장소, 배포 환경, 클라우드 이전은 추후 별도 문서로 정리합니다.

# 글로벌 작업 규칙

이 문서는 `codexAuto` 저장소의 전역 규칙입니다.

특정 프로젝트의 DB 구조, 배포 방식, 운영 기수, 서비스별 세부 내용은 이 문서에 넣지 않습니다.
실제 프로젝트 파일과 루트 폴더는 작업 시 별도로 확인합니다.

## 기본 원칙

- Codex 채팅 자체는 PC마다 공유되지 않을 수 있습니다.
- 작업 연동의 기준은 GitHub 저장소, 상태 문서, 인수인계 로그입니다.
- 작업 시작 시에는 항상 루트 문서와 마지막 인수인계 로그를 먼저 읽습니다.
- 실제 파일 작업이 필요하면 프로젝트 루트 폴더를 사용자에게 확인합니다.
- 작업 종료 시에는 대화/판단 맥락을 인수인계 로그에 요약하고 GitHub에 push합니다.
- 운영 중인 서비스는 안정성을 최우선으로 하며, 큰 구조 변경은 별도 테스트 환경에서 진행합니다.

## PC 간 공통 흐름

작업 시작:

```powershell
git pull
```

작업 종료:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\update-handoff.ps1
git add .
git commit -m "Update handoff"
git push
```

## 프로젝트 루트 확인 원칙

프로젝트 파일, 로컬 폴더, 배포 환경은 서비스마다 다를 수 있습니다.
따라서 Codex는 실제 파일을 수정하기 전에 루트를 자동 추정하지 않고 사용자에게 확인합니다.

```text
이번에 작업할 실제 프로젝트 루트가 어디인가요?
```

## 현재 관리 프로젝트

프로젝트 목록은 `docs/PROJECTS.md`를 참고하되, 실제 루트는 작업 시 별도 확인합니다.

# 운영 자동화 통합 계획

이 문서는 여러 운영 서비스를 장기적으로 하나의 업무 자동화 체계로 묶기 위한 전역 계획입니다.

개별 서비스의 DB 구조, 배포 방식, 운영 세부사항은 각 프로젝트 문서에 둡니다.

## 현재 필요한 서비스

1. 여비정산프로그램
2. QR체크 출석프로그램
3. 기수별 추첨프로그램
4. 교육신청/접수/선발/합격자 발표
5. 자격증 재발급

## 장기 방향

각 서비스를 따로 운영하기보다 아래처럼 공통 운영 데이터 기준으로 통합합니다.

```text
연도
└─ 과정/기수
   ├─ 신청
   ├─ 선발/합격
   ├─ 참여자 명단
   ├─ 출석
   ├─ 추첨
   ├─ 정산
   └─ 증명서/자격증
```

## 전역 설계 원칙

- 각 서비스는 `projects/{project-id}` 아래에 별도 문서를 둡니다.
- 공통 데이터는 중복 입력을 줄이는 방향으로 설계합니다.
- 운영 중인 서비스는 즉시 교체하지 않고, 테스트 환경에서 검증 후 이전합니다.
- 업체 전달용 산출물은 코드만이 아니라 업무 흐름, 화면 흐름, DB 초안, 권한 구조를 포함합니다.

## 향후 프로젝트화 대상

```text
projects/travel-settlement
projects/cohort-lottery
projects/education-application
projects/certificate-reissue
```
